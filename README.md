# Web Application Security Testing using DVWA and Burp Suite

## Overview
This project focuses on identifying and analyzing common web application vulnerabilities using Damn Vulnerable Web Application (DVWA) and Burp Suite.

## Objectives
- Understand web application attack surfaces.
- Identify common vulnerabilities.
- Perform manual security testing.
- Recommend remediation measures.

## Tools Used
- Kali Linux
- Burp Suite Community Edition
- DVWA
- Firefox Browser
- OWASP Testing Methodology

## Methodology

### Phase 1 – Environment Setup
- Install DVWA
- Configure Burp Suite
- Verify Proxy Interception

### Phase 2 – Web Security Testing
- SQL Injection
- Cross-Site Scripting (XSS)
- Authentication Testing
- Session Management Analysis

### Phase 3 – Analysis
- Capture HTTP Requests
- Analyze Responses
- Assess Security Impact

### Phase 4 – Reporting
- Document Findings
- Assign CVSS Scores
- Recommend Remediation

## Vulnerabilities Tested
- SQL Injection
- Reflected XSS
- Stored XSS
- Weak Authentication Mechanisms
- Session Security Issues

## Skills Demonstrated
- Web Application Penetration Testing
- Burp Suite Usage
- HTTP Traffic Analysis
- Vulnerability Assessment
- Security Reporting

## Results
Identified and demonstrated multiple web application vulnerabilities while providing remediation recommendations aligned with OWASP standards.
